# AGENTS.md

## Project Rules

Before changing code in any stage:

1. Read this file and the relevant documents completely.
2. Check the current project state and Git changes.
3. Execute only the requested stage.
4. Do not add new architecture or dependencies without explaining why.
5. Do not touch previous changes or code outside the stage scope.
6. Validate all inputs on the server.
7. Enforce all access checks in the backend.
8. Design the interface mobile-first.
9. Check English, Persian, and Arabic.
10. Check Dark, Light, LTR, and RTL.
11. Create migrations for database changes.
12. Write suitable tests for new functionality.
13. Run type check, lint, and tests before finishing.
14. Update relevant documentation.
15. Report changed files, executed tests, risks, and remaining work.

If a requirement is ambiguous or security-sensitive, ask before implementing.
