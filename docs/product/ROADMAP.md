# Roadmap

## Purpose

Track the staged build plan and release gates for the football performance platform.

## Status

Stage 2 product draft. Stage 1 scaffold is complete. Stage 2 product documentation is in progress for review.

## Stage Plan

1. Project scaffold, base files, development services, health check.
2. Product documentation.
3. Architecture and database documentation.
4. Design system documentation and UI infrastructure.
5. Internationalization, calendars, and timezone infrastructure.
6. Base database schema and migrations.
7. Authentication.
8. Marketing website.
9. Workspace and onboarding.
10. Invitations and members.
11. Players and teams.
12. Today Dashboard and Daily Routine.
13. Weekly Planner.
14. Habits and Goals.
15. Wellness and Pain Body Map.
16. Exercise Library and Training Plan Builder.
17. Coach Dashboard and reports.
18. Tasks, messages, and notifications.
19. PWA and offline-first behavior.
20. Backup and import/export.
21. Platform Admin and subscriptions.
22. Testing, server, staging, and production release preparation.

## MVP Milestones

- Scaffold and documentation foundation.
- Product scope and permissions approved.
- Architecture, database, and design system approved.
- Identity, workspace, and invitations implemented.
- Player/team management implemented.
- Daily player execution loop implemented.
- Coach decision loop implemented.
- Offline sync implemented for MVP workflows.

## Release Gates

- No real secrets committed.
- Type check, lint, unit tests, and relevant E2E tests pass.
- Backend access checks are tested.
- Workspace isolation is tested.
- Mobile-first flows are verified.
- English, Persian, Arabic and LTR/RTL behavior are verified where UI exists.
- Dark, Light, and System theme behavior are verified where UI exists.
- Offline-capable actions show correct sync states.

## Open Decisions

- Multi-owner support in MVP.
- Messaging depth in MVP.
- Assistant default access to wellness and pain records.
- Report export timing.
- Assessment default templates.
